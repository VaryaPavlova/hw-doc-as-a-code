# Архитектура системы

## Общее описание

Система построена по клиент-серверной архитектуре. Веб-клиент взаимодействует с Backend через REST API. Backend обеспечивает бизнес-логику, работу с базой данных и взаимодействие с внешними сервисами.

## Компоненты системы

```
┌─────────────┐     REST API      ┌─────────────────┐
│  Web Client │ ◄────────────────► │   Backend API   │
│   (SPA)     │                   │                 │
└─────────────┘                   └────────┬────────┘
                                           │
                          ┌────────────────┼────────────────┐
                          │                │                │
                   ┌──────▼──────┐  ┌──────▼──────┐  ┌──────▼──────┐
                   │ PostgreSQL  │  │  S3 Storage │  │ Mail Service│
                   │    (БД)     │  │  (файлы)    │  │   (SMTP)    │
                   └─────────────┘  └─────────────┘  └─────────────┘
```

## Описание компонентов

### Web Client (SPA)
Одностраничное приложение, работающее в браузере. Отвечает за отображение интерфейса и взаимодействие с пользователем.

### Backend API
REST-сервер, реализующий бизнес-логику платформы:
- Аутентификация и авторизация пользователей
- Управление материалами и специализациями
- Обработка домашних заданий
- Управление обсуждениями и вопросами

### PostgreSQL
Реляционная база данных. Хранит все сущности системы: пользователей, специализации, уроки, домашние задания, вопросы, оценки.

### S3 Storage
Объектное хранилище для файлов домашних заданий, медиаматериалов, презентаций.

### Mail Service
Внешний почтовый сервис для отправки уведомлений пользователям (например, о проверке ДЗ).
